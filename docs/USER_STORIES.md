# Client Registration

## Through first deployment

User goes to index page, sees the editor.
Can edit, click deploy.
Sees a box to enter email and subdomain. Subdomain needs to be new, email needs to be real and new.
Enters new subdomain, email. Gets told to check inbox (local gets a link to dev-email).
Clicks the link to his site in the email.
Gets redirected to a loading icon, which changes to his site. The site should match what he edited.
If the domain are used, or an email is not valid or already used, user should get a message.

## Through register button

User goes to the editor, clicks Register.
Enters email (must be valid). Gets told to check inbox (local: dev-mail).
Clicks the confirm link, lands signed in on their dashboard.
Dashboard offers claiming a site (or they can return to the editor to build/deploy).
